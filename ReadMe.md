
I、MandateTool
===========
主要实现的功能是：

> 用于通过输入框输入 *序列号* ，点击 **生成** 按钮，通过规则生成 *授权码* 输出。 

> 详细的实现流程可以参考文件 *ProcessDescription.vsdx* 中的流程说明

包含的技术亮点：

> 1、取消了标题栏并且不显示右上角的控制按键 \
> 2、随机界面的加载 \
> 3、文本框输入中提示语句的功能，动态变化是否显示 \
> 4、关闭退出软件的是界面逐渐消失的效果 \
> 5、在界面上面增加按键，实现标题栏的功能 \
> 6、可以支持随着鼠标移动窗口 \
> 7、可以支持加载动态库的功能 \
> 8、在任务栏点击图标进行界面的显示 \
> 9、防止多开，多次启动始终将第一次启动的界面显示在前

II、WindowsFormsApplication
=======================
主要实现的功能是：

> 1、获取当前主机的 *CPUID* 和 *网卡MAC地址*，通过规则生成 *序列号* ；\
> 2、判断是否在制定路径下存在固定名称的文件，不存在则弹出授权界面，并输出前面生成的 *序列号*，且要求输入 *授权码*； \
> 3、用户在授权界面可以选择 **退出**，那将会直接退出程序；\
> 4、用户在授权界面选择 **提交**，则需要正确的 **授权码** 才能进行；\
> 5、输入正确的 **授权码** 后，点击 **提交** 按钮，通过规则判断是否输入的有效，有效则判断授权通过，并且将相关信息写入到固定的文件中；\
> 6、如果当前路径下存在所规定的的文件，那么读取文件内容，并且通过规则判断文件内容是否有效；有效则判断通过，无效则弹出授权界面；\
> 7、详细的实现流程可以参考文件 *ProcessDescription.vsdx* 中的流程说明

包含的技术亮点：

> 1、获取新系统相关信息 \
> 2、C#二进制文件的读写操作 \
> 3、随机码生成 \
> 4、加密和解密的算法提供，非动态库中方法 \
> 5、以对话框的形式显示窗体
